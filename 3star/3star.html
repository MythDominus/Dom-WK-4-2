Your sketch:

Instructions

This is a very challenging exercise. You can decide whether you're ready to take it on.

Your task is to implement a (basic) clone of Tetris, which you can play online for free.

Focus on these three aspects:

  1. How to represent the Tetronimoes (the blocks). I recommend using a grid approach so you can use a 2D array to store the positions of the squares that make up a block. For example, here is the 2D array for the "T" block: 
    [
  [0, 1, 0],
  [1, 1, 1]
]
    Implement one or two of the simpler blocks first, like the "O" (the square) or the "I".
  2. Make a sequence of Tetrominoes drop from the top of the canvas to the bottom. Like Snake, which you implemented in PDM 1, you can think of the canvas as a grid of squares. The pieces should move by jumping one grid cell at a time rather than moving pixel by pixel. Allow the user to move the pieces left and right. Although it should ideally be possible to rotate the pieces as shown below, you will probably want to leave this part of the implementation for an optional extra challenge.
  3. Use collision detection to make each falling piece stop when it reaches the bottom of the canvas or its path is blocked by a previous piece.

Expected output:

The simplest and most efficient method of collision detection for Tetris is grid-based. You should have a grid representing the entire canvas and each Tetromino should also be a grid. Look back at your code (or the sample solutions) from last practical for clues.

To make the falling pieces stop when they come into contact with the bottom of the canvas or another piece, your grid class will need a method to detect if any of its occupied cells are in contact with occupied cells in another grid. This is a bit different to the grids you've seen so far and it is left for you to figure out.

If you would like to implement Tetromino rotation, do some research on how to "transpose a matrix".
